OpenGeO
Specification Manifesto GitHub

Open semantic specification

Making organisational meaning understandable to AI.

OpenGeO helps organisations publish canonical meaning for intelligent systems: who they are, what they offer, how their resources relate, and how their brand should be interpreted.

Read the spec Read the manifesto

One semantic model. Multiple projections.

OpenGeO is the semantic contract. AI Twins, HTML, JSON, APIs, MCP tools, and discovery files are projections of that contract, each serving a different consumer without changing the underlying meaning.

Circle 1 Plan

The first implementation circle turns the premise into a practical implementation sequence for organisations that need to evaluate how AI systems currently understand them.

1. Assess

Measure AI visibility and identify drift, gaps, omissions, and contradictions.

2. Declare

Generate a canonical OpenGeO semantic model and project it into AI Twin documents.

3. Test

Publish `geo.txt`, expose useful integrations, and test whether AI answers improve.

OpenGeO is

  • An open semantic specification
  • A publisher-controlled canonical representation
  • A resource-level semantic declaration model
  • A complement to `llms.txt`
  • Opt-in by design

OpenGeO is not

  • An SEO ranking algorithm
  • A moderation system
  • A proprietary merchant API
  • A replacement for `llms.txt`
  • A guarantee of objective truth
