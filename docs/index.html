Open semantic specification

Semantic Twins for the Agentic Web.

OpenGEO defines a publisher-controlled semantic layer for AI-mediated experience: canonical meaning, graph traversal, context architecture, media equivalence, and discovery through alternate links, ARD, geo.txt, and optional MCP tools.

Read the spec Read the manifesto

One semantic contract. Multiple projections.

OpenGEO is the semantic contract. Semantic Twins, HTML, geo.txt, ARD catalogs, MCP tools, APIs, and optional JSON-LD compatibility outputs are projections of that contract, each serving a different consumer without changing the underlying meaning.

OpenGEO + ARD

ARD provides the agentic discovery and trust handshake. OpenGEO provides the canonical semantic experience those discovered resources expose.

Discovery

HTML alternate links expose page-level twins; ARD catalogs and geo.txt expose broader declarations and optional tool endpoints.

Experience

Semantic Twins bind facts, canonical media, graph edges, context, and human handoff into one AI-legible projection.

Execution

MCP and API tools may add refreshed availability, faceted search, rendered cards, service availability, and other optional operations.

Context Architecture

OpenGEO defines sparse context attributes for tone, sensitivity, intent, product-surfacing policy, escalation, and human handoff. This allows the same organisation to preserve different conduct in different journeys, such as commercial discovery, advice, support, or sensitive care.

Roadmap

Circle 1

Explicit Semantic Twin projection, HTML alternate links, geo.txt, ARD discovery, canonical media, graph traversal, and optional MCP tools.

Circle 2

Passive intent and context modes inferred from resource, journey, query, session, and sensitivity signals.

Circle 3

Controlled agentic execution through first-party assistants, live tools, rendering contracts, booking, and checkout handoff.

OpenGEO is

  • An open semantic specification
  • A publisher-controlled canonical representation
  • A resource-level semantic declaration model
  • A complement to ARD, `llms.txt`, and MCP
  • Opt-in by design

OpenGEO is not

  • An SEO ranking algorithm
  • A moderation system
  • A proprietary merchant API
  • A replacement for ARD
  • A replacement for `llms.txt`
  • A requirement to use JSON-LD
OpenGEO is published by Zahid Saleem. Built in alignment with the open-agentic web movement, including Jeremy Howard and Answer.AI's `llms.txt` proposal and emerging Agentic Resource Discovery patterns.
